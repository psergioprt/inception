    Select Insert Guest Additions CD image.

    Click on the new CD icon that appeared on the left, and click Run Software.

    sudo reboot

Build Inception

    To implement the Inception project, you will need three services, each configured in a separate Docker container. These containers will have to be connected to a single isolated network, allowing exclusive communication between them. Communication with the outside world will be restricted to port 443 and exclusively through the container running the Nginx service.

    The directory structure should be organised as specified in the project statement in order to improve the organisation and scalability of the project. Below is a representation of the suggested directory structure:

.
├── Makefile
├── secrets
│   ├── credentials.txt
│   ├── db_password.txt
│   └── db_root_password.txt
└── srcs
    ├── docker-compose.yml
    ├── .env
    └── requirements
        ├── mariadb
        │   ├── Dockerfile
        │   ├── .dockerignore
        │   └── conf
        ├── nginx
        │   ├── Dockerfile
        │   ├── .dockerignore
        │   └── conf
        └── wordpress
            ├── Dockerfile
            ├── .dockerignore
            └── conf

    This structure facilitates maintenance and allows the project to be scalable, clearly separating the components and their respective configurations.

    For better control and visualisation of the organisation of the directory structure, we recommend installing the tree program on your virtual machine (VM).

    Install Tree Command

        Within the VM, open the terminal and update your existing package list with the command:

        sudo apt update

        Still in the terminal, install the tree package with the command:

        sudo apt install tree

In the terminal, check if the installation was successful:

tree --version

    From now on, whenever you want to see the directory structure, you can run the tree or tree -a command inside the parent directory.

#Initial Configuration for NGINX

#    Create the compose file docker-compose.yml in the path described below via the terminal:
mkdir -p rootInception/srcs/ && nano rootInception/srcs/docker-compose.yml

# Fill in the docker-compose.yml so that when executed, it can compose the NGINX container as shown below:
version: "3.8"

services:
  nginx:
    build: requirements/nginx/.
    container_name: nginx
    ports:
      - "443:443"                # HTTPS required by the subject, not 80:80
    restart: unless-stopped      # Subject requires 
    networks:                    # Subject requires
      - inception-network

networks:                        # define the network
  inception-network:
    driver: bridge

# Only port 443 (HTTPS) should be exposed to the outside world.
# Nginx is the entry point for all traffic, and TLSv1.2/1.3 is mandatory.
# Exposing port 80 is not required and may be considered non-compliant during evaluation.

# restart: unless-stopped → The subject explicitly says:
# Your containers must restart automatically in case of a crash.” ✅

# Networks → The subject says:
# “A docker-network that establishes the connection between your containers.”

version: "3.8" in docker-compose.yml specifies the Compose file format version.

3.x versions are compatible with modern Docker and Swarm features.

3.8 is a stable version that supports:

Networks

Volumes

Restart policies

Build options

💡 TL;DR: Using 3.8 ensures all features needed for Inception (networks, volumes, restart) work correctly.

You could use 3.9 too, but 3.8 is widely used and safe for this project.

RUN apt update && apt upgrade -y && apt install -y nginx && rm -rf /var/lib/apt/lists/*

ENTRYPOINT ["nginx", "-g", "daemon off;"]

docker compose up --build 

# For ports 443 HTTPS, it needs TLS and blocks configuration, otherwise cannot access localhost (with ports 80 it can)

# 1 - Create a folder for your certificates
# Inside your NGINX config directory:
mkdir -p rootInception/srcs/requirements/nginx/conf/ssl
cd rootInception/srcs/requirements/nginx/conf/ssl

# 2 - Generate a self-signed certificate
openssl req -x509 -nodes -days 365 -newkey rsa:2048 -keyout nginx.key -out nginx.crt -subj "/C=PT/ST=Lisbon/L=Lisbon/O=42/OU=Inception/CN=pauldos-.42.fr"

# 3 - Configure Nginx server block for HTTPS
# Create/edit default.conf inside conf folder:

#FOR 42 EVALUATION
server {
    listen 443 ssl;
    server_name localhost;

    ssl_certificate /etc/nginx/ssl/nginx.crt;
    ssl_certificate_key /etc/nginx/ssl/nginx.key;
    ssl_protocols TLSv1.2 TLSv1.3;

    root /var/www/html;
    index index.html;
}


#JUST FOR A REAL LIFE SERVER CONFIG

You buy a domain name (e.g., from Namecheap or Google Domains).

You point the domain’s DNS A record to your server’s public IP.

In your Nginx config, you set

Then you’d use Let’s Encrypt or another CA to generate a trusted TLS certificate (no browser warning).
